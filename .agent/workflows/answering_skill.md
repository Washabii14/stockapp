# Answering Skill

When answering questions about the `my_stocks_analysis` project, operate under these principles:

## 1. Ground Truth First
- Always consult the project's source code, `README.md`, and existing documentation before making assumptions about how the application works.
- Ensure any technical answer accurately reflects the actual tech stack in use (e.g., the specific framework, charting library, or API provider used in this project).

## 2. Technical Precision
- Provide technically exact answers tailored to this web application's context. Avoid generic programming advice; tailor answers to the specific libraries, patterns, and data domain (stocks/finance).
- Explain *why* a solution works within this framework and how it relates to the financial data domain.

## 3. Scope Restriction
- Adhere strictly to the requested question or task. Do not offer unsolicited grand refactors or architecture overhauls unless they are direct, immediate blockers or present a severe correctness, security, or performance threat.
- If a question touches on financial advice (e.g., "should I buy this stock?"), clarify that this tool is for analysis and information only—not financial advisory.

## 4. Clear Structure
- Format answers clearly using Markdown headers, bullet points, and code blocks where appropriate.
- Provide step-by-step reasoning or explanations for complex topics.
- If providing a script or code snippet, ensure it is directly compatible with the current project's setup (tech stack, file structure, environment).
